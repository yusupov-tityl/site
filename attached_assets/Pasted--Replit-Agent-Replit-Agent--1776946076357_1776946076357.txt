# Промт для Replit Agent — правки после ревью

Контекст: сайт **itityl.ru** (Ай-Титул), Vite + React + TS + Tailwind v4 + framer-motion + wouter. Страницы `/products`, `/technologies`, `/services` уже созданы. Ниже — список правок по результатам ревью. Исправь всё пунктом-к-пункту.

## 1. Хлебные крошки (breadcrumbs)

На всех внутренних страницах, **кроме главной**, добавить хлебные крошки в верхней части контента — **под `SiteNav`, до `PageHero`**:

- `/services` → «Главная — Услуги»
- `/products` → «Главная — Продукты»
- `/technologies` → «Главная — Технологии»
- детальные (`/services/*`, `/products/*`) → «Главная — <Раздел> — <Название>»

Создать компонент `Breadcrumbs.tsx` (принимает `items: { label: string; href?: string }[]`). Последний элемент — текущая страница без ссылки. Стилистика: мелкий `text-xs uppercase tracking-widest text-white/50`, разделитель «—», ссылки с hover на `#FFE600`. Добавить `schema.org` разметку `BreadcrumbList` (JSON-LD) для SEO.

## 2. Адаптивность — проверить на всех разрешениях

Есть жалоба, что на **MacBook Air 13" (1440×900)** блок не помещается в экран при 100% масштабе. Пройди всю вёрстку и проверь:

- Breakpoints: `sm` 640, `md` 768, `lg` 1024, `xl` 1280, `2xl` 1440.
- Hero-секции — H1 не должен вылезать за viewport на 1440×900. Уменьши клемпы шрифтов для `xl` если нужно: например `text-[clamp(2.5rem,5vw,4.5rem)]` вместо фиксированных 72px.
- Grid-карточки 2×2, 3×2, 5 колонок — на средних экранах перестраивать в 2 или 3 колонки, не держать 5 колонок ниже `2xl`.
- `max-w-[1440px] mx-auto px-6` — проверь горизонтальные переполнения.
- Протестируй мысленно: 1280, 1366, 1440, 1920, 375 (iPhone SE), 390 (iPhone 14), 768 (iPad).

## 3. Чёрные кнопки — починить читаемость текста

На нескольких кнопках **не виден текст** (кнопка + текст одинакового цвета). Проблема:
- В финальной форме контактов (кнопка submit).
- Некоторые CTA-кнопки внутри страниц.

Пройди по всем `<button>` и CTA-компонентам. Исправь: если фон `bg-black` — текст должен быть `text-white` или `text-[#FFE600]`; если фон `bg-[#FFE600]` — текст `text-black`. На hover — инвертированный вариант с явным контрастом. Проверь disabled-состояния.

## 4. Кнопки CTA → попап с формой вместо скролла к форме

Сейчас CTA-кнопки («Заказать консультацию», «Обсудить проект», «Получить консультацию», «Связаться с командой» и т. д.) скроллят к форме внизу страницы. Это плохо по двум причинам:

1. Клиент теряет место чтения.
2. Нельзя поставить отдельные счётчики аналитики на каждую кнопку.

**Переделай:** все CTA-кнопки открывают **модалку (попап) с `ContactForm`** поверх страницы.

- Создать `ContactModal.tsx` — полноэкранный overlay (dim `bg-black/70 backdrop-blur`), центрированное окно `max-w-[640px]`, крестик закрытия, закрытие по Escape и клику вне окна, focus-trap.
- Создать `useContactModal` (Zustand или просто React context) — глобальный стейт `isOpen`, `source` (откуда открыта).
- `source` прокидывается в форму как hidden-поле `data-source` → уйдёт в API, это нужно для аналитики.
- Каждая CTA-кнопка принимает проп `ctaSource: string` (например `"products-hero"`, `"products-faq"`, `"services-card-development"`) и при клике вызывает `openModal(source)`.
- Финальную секцию с формой в конце каждой страницы **оставить** (там форма inline), но все остальные кнопки ведут в модалку.
- Добавить `data-analytics` атрибуты на каждую кнопку для будущей подстановки счётчиков.

## 5. Переходы между разводящими — скролл к началу страницы

Кнопки-переходы:
- `/products` блок 1.8 «Перейти к услугам» → сейчас ведёт в **произвольную точку** страницы `/services`.
- `/services` блок 3.7 «Перейти к продуктовым направлениям» → та же проблема на `/products`.

Причина — скорее всего hash/якорь или отсутствие `scrollTo(0, 0)` при навигации. Исправить: при переходе между страницами всегда скроллить окно в начало. Сделать `ScrollToTop` компонент, монтируемый в корне роутера: слушает изменение `location` (wouter `useLocation`) и вызывает `window.scrollTo({ top: 0, behavior: "instant" })`. Убедись, что Lenis (SmoothScroll) не конфликтует — если нужно, прокидывай `lenis.scrollTo(0, { immediate: true })`.

## 6. Навигация `/technologies` из главного меню — ведёт в FAQ

При клике на пункт «Технологии» в `SiteNav` сайт открывает страницу и **сразу скроллит к блоку «Частые вопросы»** вместо hero. Вероятные причины:
- В ссылке `href="/technologies#faq"` вместо `/technologies`.
- Или focus-management автоматически переводит в первый `<details>`/`<summary>`.

Проверь `SiteNav.tsx`: все основные пункты (`/products`, `/technologies`, `/services`) должны вести на чистый путь без хеша. После `ScrollToTop` из пункта 5 проблема должна уйти автоматически, но перепроверь.

## 7. Логотип — клик ведёт на главную

Логотип в шапке (`SiteNav` / `SiteFooter`) сейчас, судя по правкам, не ведёт на `/`. Оберни логотип в `<Link href="/">` с `aria-label="На главную"` и `data-cursor="link"`.

## 8. Мобильное меню

В мобильной версии (< `md`, 768px) **меню не отображается вообще**. Нужно:

- В `SiteNav.tsx` добавить мобильный гамбургер-бургер (иконка `Menu` из `lucide-react`) справа в шапке, видимый только на `< md`.
- При клике — открывается полноэкранный drawer (слайд-ин справа или сверху) с ссылками на `/`, `/products`, `/technologies`, `/services`, `/about` (если есть), контактами.
- Закрытие: крестик (`X`), свайп, Escape.
- Фон `bg-[#0A0A0A]`, крупные ссылки (`text-2xl font-heading`), разделители.
- При переходе по ссылке — drawer закрывается.
- Блокировать скролл body когда открыто (`document.body.style.overflow = "hidden"`).
- Анимация через framer-motion (`AnimatePresence` + slide).

## 9. Финальная проверка

После всех правок:
1. `pnpm typecheck` — 0 ошибок.
2. `pnpm build` — успешно.
3. Ручная проверка каждой страницы: навигация из меню, breadcrumbs, все кнопки (читаемость + попап), переходы между разводящими, мобильное меню 375px, десктоп 1440×900 без переполнений.
